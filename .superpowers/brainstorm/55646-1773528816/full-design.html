Full Design: Auto-Icon Fiche Cards

Complete proposal — review each section

1. Data Model

Add two columns to the fiches table:

  • icon — string, nullable — Lucide icon name (e.g., music, flower-2, heart)
  • icon_color — not stored, computed from id % 6 at render time

When icon is null, the fallback document icon is shown (backwards-compatible).

2. Automatic Icon Assignment

Fiche saved → Dispatch job → Claude API picks icon → Store in DB

Trigger: When a fiche is created or its title changes, a queued job (AssignFicheIcon) is dispatched.

The AI prompt: Send the fiche title (+ description if available) along with the curated list of ~100 pre-selected Lucide icon names relevant to care activities. Claude returns one icon name.

Why a curated subset? Lucide has 1700+ icons, but many are tech/dev-oriented (git-branch, terminal, database). A curated list of ~100 relevant icons ensures the AI never picks something weird. Examples:

music heart flower-2 palette cooking-pot scissors puzzle sun bird cake camera book-open trees dog star smile gift party-popper brain footprints

Fallback: If the AI call fails, icon stays null and the document icon is shown. A retry can be triggered manually or via scheduled command.

Backfill: An artisan command php artisan fiches:assign-icons processes all existing fiches that have no icon set. Run once after deployment.

3. Deterministic Color Palette

6 colors, assigned by fiche->id % 6. Same fiche always gets the same color.

Orange
Teal
Yellow
Purple
Green
Rose

These extend the existing 4 avatar colors (orange, teal, yellow, purple) with green and rose. Same pastel-bg + saturated-icon pattern.

4. UI Changes

The fiche list item gets a visual upgrade:

  • Icon disc: 32×32px square → 48px circle with colored background
  • Title: 16px → 18px, semibold
  • Padding: 16px → 18px vertical, 20px horizontal
  • Gap between items: 8px (unchanged, already good)

5. Components Affected

  • New: <x-fiche-icon> — Blade component that renders the colored disc + Lucide icon. Takes :fiche prop, computes color from ID, renders icon by name.
  • Modified: fiche list in initiatives/show.blade.php — replace inline SVG with <x-fiche-icon>, adjust spacing/sizing classes.
  • New: AssignFicheIcon job — queued job that calls Claude API to pick icon.
  • New: fiches:assign-icons artisan command — backfills icons for existing fiches.
  • Migration — add icon string column to fiches table.
  • Lucide package — install blade-lucide-icons for Blade rendering.

6. Out of Scope

  • No manual icon picker UI — AI handles it
  • No icon on initiative cards (only on fiche list items within an initiative)
  • No changes to fiche detail page (just the list view)
  • No dark mode variants
